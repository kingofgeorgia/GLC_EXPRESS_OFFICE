# GLC EXPRESS OFFICE

CRM-система для транспортной компании на ASP.NET Web Forms (.NET Framework 4.8).

Сейчас проект включает:

- авторизацию через форму входа;
- вкладку `Рейсы` с созданием, редактированием и удалением рейсов;
- вкладку `Водители` с хранением контактных данных и загрузкой сканов документов;
- вкладку `Автопарк` с привязкой существующих водителей к автомобилям;
- вкладку `Клиенты` с карточками клиентов;
- сохранение данных в `App_Data/crm-data.json`.

## Стек

- ASP.NET Web Forms
- .NET Framework 4.8
- Bootstrap
- Newtonsoft.Json

## Структура CRM

### Рейсы

Поля:

- `№`
- `Клиент`
- `Статус`
- `Страна`
- `Автомобиль`
- `Водитель`
- `Дата начала`
- `Дата окончания`
- `Фрахт`
- `Предоплата`

### Водители

Поля:

- `ФИО`
- `Дата рождения`
- `Номер телефона`
- `Адрес проживания`
- `Скан копия паспорта`
- `Скан копия водительского удостоверения`

### Автопарк

Поля:

- `Марка автомобиля`
- `Модель автомобиля`
- `Гос. номер`
- `VIN код`
- `Марка прицепа`
- `Модель прицепа`
- `Гос. номер прицепа`
- `Скан копия документов`
- `Закрепленные водители`

### Клиенты

Поля:

- `Клиент`
- `Направление`
- `Менеджер`
- `Телефон`
- `Эл. почта`

## Что уже реализовано

- CRUD-операции во всех вкладках CRM
- связность данных между клиентами, водителями, автопарком и рейсами
- корректное обновление связанных записей при редактировании
- очистка связанных ссылок при удалении записей
- загрузка файлов для документов водителей и автопарка
- защита страницы CRM через авторизацию

## Запуск

### Вариант 1: через Visual Studio

1. Откройте `GLC_EXPRESS.sln`.
2. Восстановите NuGet-пакеты.
3. Запустите проект через IIS Express.

### Вариант 2: через MSBuild

Понадобятся:

- .NET Framework 4.8 Developer Pack
- IIS Express или Visual Studio Web Build Tools
- восстановленные NuGet-пакеты

После сборки откройте страницу CRM:

- `http://localhost:52722/orders`

## Авторизация

Страница входа:

- `/Login`

Настройки пользователей сейчас находятся в `Web.config`.

Перед публикацией в production рекомендуется:

- заменить тестовые учетные записи;
- вынести пользователей в базу данных;
- не хранить постоянные пароли в конфигурации приложения.

## Данные и файлы

- основное хранилище: `App_Data/crm-data.json`
- загружаемые файлы: папка `Uploads/`

Папка `Uploads/` в репозиторий не коммитится.
